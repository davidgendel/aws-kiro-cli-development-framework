# Feature Specs

This folder contains feature-level specifications. Each spec describes a single
feature or capability — what it does, who it's for, and what "done" looks like.

Specs serve two purposes:
1. Alignment — ensure the team agrees on what is being built before building it
2. AI context — Kiro can reference specs to generate accurate, scoped implementations

---

## How to Write a Spec

- One file per feature, named descriptively: `user-authentication.md`, `export-to-csv.md`
- Keep specs outcome-focused, not implementation-focused
- Define acceptance criteria clearly — these become the basis for tests
- Link to relevant tenets or strategy goals where applicable

See `features/example-feature.md` for a full example.
